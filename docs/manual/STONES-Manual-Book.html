Bab 1

Pendahuluan — Apa itu STONES

STONES (Business Process Suite) adalah aplikasi web untuk mengembangkan, mengelola, dan menyimpan Business Process (BP), Standard Operating Procedure (SOP), dan dokumen perusahaan dalam satu tempat — lengkap dengan alur persetujuan (approval), riwayat versi, impor PDF, generator diagram, dan asisten AI yang bisa membaca basis pengetahuan internal Anda.

Yang bisa Anda lakukan dengan STONES

  • Membuat Business Process Map otomatis hanya dari satu tabel SIPOC.
  • Menyusun matriks tanggung jawab RASCI yang terisi otomatis.
  • Mengubah PDF SOP lama menjadi dokumen terstruktur dengan bantuan AI.
  • Membuat flowchart swimlane, diagram taksonomi, dan value chain tanpa menggambar manual.
  • Menjalankan alur persetujuan dokumen dengan versi & jejak audit.
  • Bertanya & menganalisa proses lewat Ask AI.

Apa yang Anda butuhkan

KebutuhanKeterangan
Peramban (browser) modernChrome, Edge, atau Firefox versi terbaru.
Akun GoogleUntuk login — akses dikunci hanya untuk akun yang diizinkan.
Koneksi internetData disimpan & disinkronkan di cloud secara realtime.
API key AI (opsional)Diperlukan hanya untuk fitur Ask AI dan Document Import. Gratis dari OpenRouter — lihat Bab 13.

Bab 3

Konsep Dasar

Beberapa prinsip yang berlaku di seluruh aplikasi. Memahaminya sekali akan memudahkan Anda memakai semua menu.

Autosave — tanpa tombol "Simpan"

Hampir semua editan tersimpan otomatis beberapa saat setelah Anda mengetik. Anda tidak perlu menekan tombol simpan. Khusus Business Process, gunakan Save version bila ingin membuat snapshot yang bisa dipulihkan (restore) nanti.

Realtime & kolaboratif

Data disimpan di cloud dan disinkronkan realtime. Perubahan dari rekan atau dari perangkat lain langsung muncul di layar Anda tanpa perlu refresh.

Tipe dokumen (docType)

Setiap dokumen memiliki tipe yang menentukan menu tempat ia dibuka. Di Repository, tipe ditampilkan sebagai chip berwarna.

TipeDibuat di menu
BPDocument Development
SOPDocument Import
FLOWAuto Flow Process
TAXONOMY / HLP / TAXDESCTaxonomy Suite
KNOWLEDGEAI Knowledge Base (tidak tampil di Repository)

Status & alur persetujuan dokumen

Business Process mengikuti alur status berikut:

Draft  →  In Review  →  Approved  →  Published

Dokumen berstatus In Review masuk ke antrian Document Action Request untuk di-approve atau reject. Setiap perubahan tercatat di riwayat versi dan audit trail.

Bab 5

🏗️ BP Architecture

Mengelola hierarki Business Process multi-entitas — dari korporasi (Level 0) hingga proses paling rinci (Level 3) — dalam satu pohon yang bisa dijelajahi.

Struktur level

LevelMakna
LVL 0Entitas / korporasi tersendiri.
LVL 1Kategori proses di dalam entitas.
LVL 2Kelompok proses.
LVL 3Proses rinci (daun/leaf) — di sinilah SIPOC, Risk, dan Performance Indicator disimpan.

Tata letak layar

  • Panel kiri — navigator pohon (tree) yang bisa difilter per entitas.
  • Panel kanan — editor untuk node yang dipilih: kode, judul, SIPOC, daftar Risk, dan Performance Indicator (KPI).

Pada baris SIPOC, kolom Supplier/Customer bertipe Process dapat ditautkan (link) ke node lain — bahkan lintas entitas — sehingga keterhubungan antar-proses terjaga.

Tip: Gunakan tombol seed sample untuk memuat contoh pohon proses sebagai titik awal, lalu sunting sesuai organisasi Anda.
Bab 6

✏️ Document Development — Studio Utama

Menu inti STONES: membangun Business Process lengkap hanya dari satu tabel SIPOC. Terdapat tiga tampilan yang bisa dipindah lewat tab di bagian atas.

Tampilan 1 — SIPOC Editor

Isi baris demi baris, di mana 1 baris = 1 hubungan:

KolomArti
SupplierPihak yang memberi masukan.
InputMasukan yang diberikan.
ProcessAktivitas / sub-proses.
OutputKeluaran yang dihasilkan.
CustomerPihak yang menerima keluaran.
  • Tambah / hapus baris sesuai kebutuhan.
  • Paste langsung dari Excel — salin blok sel, tempel ke tabel.
  • Editor PPI (Process Performance Indicator) untuk indikator kinerja.
  • Impor data dari file .xlsx / .csv.

Tampilan 2 — Business Process Map

Diagram yang digambar otomatis dari SIPOC: kotak proses, aktor (supplier/customer), panah in/out/handoff, dan title block ITM (logo, Prepared/Reviewed/Approved, BP No, revisi).

AksiCara
Edit header / logoKlik field header atau logo lalu ubah.
Atur posisi kotakDrag kotak ke tempat yang diinginkan.
Ganti nama kotakDouble-click pada kotak.
Rapikan otomatisKlik Auto-arrange.
Label garisToggle Flow # / Text.
EksporExport PNG (persis tampilan web) atau Export JSON.
Tahu tidak? Garis antar-kotak dirutekan otomatis agar menghindari kotak lain, dan langsung dirapikan ulang setiap kali Anda menggeser sebuah kotak.

Tampilan 3 — RASCI

Matriks tanggung jawab: baris = sub-proses, kolom = aktor. Terisi otomatis dari SIPOC dan bisa Anda override per sel. Ekspor ke CSV atau XLSX.

HurufPeran
RResponsible — yang mengerjakan.
AAccountable — yang bertanggung jawab akhir.
SSupport — yang membantu.
CConsulted — yang dimintai pendapat.
IInformed — yang diberi informasi.

Document control

Setiap Business Process dilengkapi:

  • Status: Draft → In Review → Approved → Published.
  • Riwayat versi (snapshot + restore) dan audit trail.
  • Komentar melalui panel samping.
  • Lampiran file (PDF/PNG) dan deep-link untuk berbagi (?doc=BP-xxxx).

Bab 7

📥 Document Import — PDF menjadi Data Terstruktur

Mengubah PDF SOP / BP / policy lama menjadi dokumen terstruktur dengan bantuan AI, sehingga tidak perlu mengetik ulang.

Perlu API key. Fitur ini memakai AI, jadi butuh API key (OpenRouter atau Google Gemini) yang diisi di kotak pada layar ini. Lihat Bab 13 untuk cara memperolehnya. Ada juga pemilih Model di kanan atas — defaultnya model gratis yang mampu membaca hasil scan.
  1. Klik atau drag file PDF (maksimal ±7 MB; hasil scan pun bisa dibaca).
  2. AI membaca dan mengekstrak isinya — biasanya 20–60 detik.
  3. Muncul layar review berdampingan (PDF di kiri, form di kanan): metadata, tujuan/scope, aktor, langkah prosedur, RASCI, dan PPI. Koreksi bagian yang kurang tepat.
  4. Klik Simpan ke Repository — tersimpan sebagai dokumen tipe SOP, dengan PDF asli ikut dilampirkan.
Selalu review. Hasil ekstraksi AI adalah data terstruktur — periksa ketepatannya sebelum menyimpan, khususnya untuk PDF hasil scan.
Bab 8

🔀 Auto Flow Process — Generator Flowchart SOP

Membuat flowchart swimlane (satu kolom per pihak yang bertanggung jawab) secara otomatis. Anda cukup mengisi data, sistem yang menggambar sesuai template.

Cara pakai

  1. Isi Kepala dokumen: judul, level, BP No, tanggal, revisi, Prepared/Reviewed/Approved, dan logo. Bagian ini bisa dilipat agar ringkas.
  2. Isi Judul section — nama proses, misalnya "C3.2 Fuel Supply".
  3. Isi Lane / kolom — satu pihak per baris, diurutkan dari kiri ke kanan.
  4. Isi tabel Langkah: No, Tipe (Start/End, Process, Sub Process, Decision, dll), Lane, RASCI, Ref, Aktivitas, dan Next.
  5. Lihat hasilnya di panel Preview di sebelah kanan secara langsung.

Aturan kolom "Next"

Isi NextArti
(kosong)Otomatis lanjut ke langkah berikutnya.
6:Yes, 3:NoPercabangan keputusan: ke langkah 6 bila "Yes", ke langkah 3 bila "No".

Tips di preview

  • Drag kotak untuk atur posisi manual; double-click untuk mengganti nama aktivitas.
  • Rapikan ulang mengembalikan semua kotak ke posisi otomatis.
  • Hilangkan centang Kepala dokumen untuk menyembunyikan title block.
  • Export PNG untuk hasil siap tempel.
  • Load sample menampilkan contoh "C3.2 Fuel Supply" sebagai titik awal.

Flow tersimpan sebagai dokumen tipe FLOW dan bisa dibuka lagi untuk disunting.

Bab 9

🗂️ Taxonomy Suite

Tiga menu generator diagram yang berbagi pola sama: form di kiri, preview langsung di kanan, autosave, tombol Load sample, dan Export PNG. Semua hasilnya muncul di Repository dengan chip tipe masing-masing.

9.1 Business Process Taxonomy

Diagram hierarki L0 → L3: rail label level, dua band membentang (L0 core process dan L1 grup), lalu grid kolom — setiap kolom berisi satu kotak kategori L2 dengan tumpukan L3 di bawahnya. Tiap kotak punya kode, nama, dan opsi highlight (garis tepi biru). Tersimpan sebagai tipe TAXONOMY.

9.2 High Level Process

Diagram value chain perusahaan: band bertumpuk (Management / Core / Enabler), tiap band berupa tray abu-abu berisi kotak proses (badge bulat kode + nama miring), dengan opsi highlight. Dilengkapi judul, subjudul, dan footnote/legenda. Tersimpan sebagai tipe HLP.

9.3 Taxonomy Description

Tabel deskripsi proses: header biru berisi nomor proses (kolom), dengan baris atribut tetap — name, description, KPI, responsible, accountable. Atribut KPI/responsible/ accountable yang bernilai banyak ditampilkan sebagai daftar bullet. Tersimpan sebagai tipe TAXDESC.

Alur kerja umum: klik Load sample untuk melihat contoh → sunting datanya di form kiri → pantau preview kanan → Simpan ke Repository → Export PNG saat siap dipakai.
Bab 10

📝 Document Action Request — Antrian Review

Pusat persetujuan dokumen. Semua dokumen berstatus In Review berkumpul di sini untuk ditindaklanjuti.

  • Klik Approve untuk menyetujui dokumen (status naik ke Approved).
  • Klik Reject untuk mengembalikan dokumen ke pembuatnya.
  • Gunakan tombol + New BP untuk memulai Business Process baru dari sini.

Alur ini melengkapi status dokumen yang dijelaskan di Bab 3.

🗄️ Repository & 🔍 Global Search

Repository

Daftar semua dokumen yang tersimpan — menampilkan ID, nama, versi, status, dan waktu terakhir diperbarui, beserta chip tipe (BP/SOP/FLOW/dll). Untuk tiap dokumen Anda bisa:

AksiHasil
OpenMembuka dokumen di menu yang sesuai tipenya.
DuplicateMenggandakan dokumen sebagai salinan baru.
DeleteMenghapus dokumen.

Dokumen referensi (AI Knowledge Base) sengaja tidak ditampilkan di Repository.

Global Search

Mencari lintas seluruh dokumen: nama, ID, isi SIPOC, aktor, flow, PPI, header, hingga komentar. Klik salah satu hasil untuk langsung membuka dokumennya.

Tip: Global Search berguna untuk menemukan proses yang menyebut aktor atau istilah tertentu meski Anda lupa nama dokumennya.
Bab 12

✦ Ask AI & 📚 AI Knowledge Base

Ask AI

Chat untuk bertanya, meringkas, dan menganalisa Business Process Anda — misalnya menemukan bottleneck, gap, kejelasan ownership, PPI yang lemah, dan rekomendasi perbaikan.

AI membaca semua BP tersimpan ditambah referensi aktif di AI Knowledge Base. Jawaban mengikuti bahasa pertanyaan (Indonesia atau Inggris).

  • Setup sekali: masukkan API key dan pilih Model (lihat Bab 13).
  • Makin lengkap SIPOC/PPI/SOP di dokumen, makin dalam analisa yang bisa diberikan AI.

AI Knowledge Base

Menyediakan bahan tambahan agar jawaban Ask AI lebih relevan dengan konteks perusahaan Anda.

  1. Beri judul referensi.
  2. Upload PDF & ekstrak (AI membaca isinya, scan pun bisa) — atau langsung paste / ketik teks.
  3. Review atau sunting teksnya, lalu klik Tambah ke knowledge base.
  4. Tiap referensi punya toggle aktif/nonaktif — hanya yang aktif yang dipakai AI. Bisa juga di-Edit atau Hapus.

Setelah itu, buka Ask AI dan bertanya seperti biasa — jawaban otomatis mempertimbangkan referensi yang aktif.

Bab 13

🔑 Setup AI — API Key & Pemilih Model

Fitur Ask AI dan Document Import memakai penyedia AI yang dipanggil langsung dari peramban Anda. Anda perlu memasukkan API key sendiri (BYO — Bring Your Own key) satu kali.

Privasi: API key disimpan hanya di peramban ini (localStorage) dan tidak pernah dikirim ke mana pun selain penyedia AI yang Anda pilih. Key tidak disimpan di server aplikasi maupun di dalam kode.

Langkah setup (contoh OpenRouter)

  1. Buat key gratis di openrouter.ai/keys.
  2. Di STONES, buka Ask AI (atau Document Import). Pada kotak API key di atas, pilih penyedia lalu tempel key Anda, dan klik Simpan.
  3. Pilih Model di kanan atas. Tersedia opsi gratis (bertanda :free).
  4. Mulai bertanya atau mengimpor PDF.

Penyedia yang didukung

PenyediaCatatan
OpenRouterDirekomendasikan; mendukung banyak model & pembaca PDF. Ada model gratis.
Google GeminiMendukung baca PDF/scan langsung.
Groq / OpenAIUntuk chat; tidak mendukung baca PDF di Document Import.
Custom (OpenAI-compatible)Isi base URL sendiri.
Kena limit? Model gratis memiliki batas pemakaian per model. Bila tercapai, cukup ganti model dari dropdown, atau top-up saldo. Saran: set limit kredit di dashboard penyedia agar biaya terkendali.
Dokumen sensitif? Sebagian penyedia free tier dapat memakai data untuk pelatihan. Untuk dokumen sangat sensitif, pilih model / endpoint berbayar dengan kebijakan no-logging.

Bab 15

❓ Tips & Pertanyaan Umum (FAQ)

Apakah perlu menekan tombol simpan?

Tidak. Editan tersimpan otomatis. Untuk BP, gunakan Save version bila ingin membuat snapshot yang bisa dipulihkan.

Apakah bisa kolaborasi realtime?

Bisa. Perubahan dari perangkat atau orang lain langsung muncul di layar Anda.

Sudah diperbarui tapi tampilan belum berubah?

Muat ulang secara paksa dengan Ctrl + Shift + R. Bila baru saja dideploy, tunggu ±1–2 menit.

Apakah PDF hasil scan bisa diimpor?

Bisa — AI mampu membacanya. Namun hasilnya berupa data terstruktur, jadi selalu review sebelum menyimpan.

Ask AI menampilkan error atau tidak menjawab?

Pastikan API key sudah diisi dan model sudah dipilih (Bab 13). Bila kena rate limit, ganti model dari dropdown.

Seberapa aman datanya?

Akses dikunci login Google + Security Rules. Diagram dan dokumen tersimpan aman di cloud; API key AI hanya tersimpan di peramban Anda.

⚙️ Untuk Admin / Pengelola

Bagian ini ditujukan untuk pengelola teknis aplikasi.

TugasPerintah / Cara
Deploy webnpm run build lalu npm run deploy (publish dist/ ke branch gh-pages).
Menjalankan lokalnpm install → npm run dev (buka http://localhost:5173/bp_msbp/).
Akses / keamananFirebase Auth (Google sign-in) + Security Rules (request.auth != null).
DataFirestore collection bp_documents; lampiran di Cloud Storage.
Catatan. Fitur AI kini dipanggil langsung dari peramban (client-side) memakai API key milik pengguna — tidak memerlukan deploy Cloud Functions. Perubahan frontend cukup npm run deploy.

Keamanan API key Firebase

Firebase Web apiKey yang ada di kode adalah key publik client (by design). Proteksi sesungguhnya adalah Security Rules + Auth, ditambah API key restriction di GCP. Jangan pernah menaruh API key AI yang sensitif di dalam kode/bundle.
